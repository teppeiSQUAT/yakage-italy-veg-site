# SCSS 開発環境ワークフロー

- **作成日**: 2025年3月3日
- **対象**: yakage-italy-veg テーマ

---

## 概要

テーマのスタイルは SCSS で記述し、コンパイル後に `style.css` が生成される。編集後に自動コンパイルし、ブラウザで確認できる。

---

## ディレクトリ構成

```
wordpress/html/wp-content/themes/yakage-italy-veg/
├── scss/
│   ├── style.scss       # エントリーポイント（テーマヘッダー含む）
│   ├── _variables.scss  # CSS 変数
│   ├── _reset.scss      # リセット・ベース
│   ├── _layout.scss     # コンテナ・セクション
│   ├── _components.scss # ボタン等
│   ├── _header.scss     # ヘッダー
│   └── _footer.scss     # フッター
└── style.css            # コンパイル出力（Git 管理）
```

---

## コマンド

| コマンド | 用途 |
|----------|------|
| `npm run sass:build` | 1回だけコンパイル |
| `npm run sass:watch` | ファイル変更を監視し、自動コンパイル |

---

## 開発環境の起動手順

1. **Docker 起動**（WordPress 開発環境）
   ```bash
   docker compose up -d
   ```

2. **SCSS 監視開始**（別ターミナル）
   ```bash
   npm run sass:watch
   ```

3. **ブラウザで確認**
   - http://localhost:8080 を開く
   - SCSS を編集 → 自動コンパイル → ブラウザをリロードして確認

---

## 注意事項

- **編集対象**: `scss/` 内の `.scss` ファイルを編集する。`style.css` は直接編集しない（上書きされる）
- **style.css**: コンパイル後に Git にコミットする（本番環境ではこのファイルが読み込まれる）
