# フェーズ0 実施手順書

- **作成日**: 2025年3月2日
- **目的**: 本マシン・別マシンで同じ手順で開発環境を揃える

## 前提

- Git がインストール済みであること
- **初回コミット前に** `git config user.name` と `git config user.email` を設定すること（リポジトリ内: `git config user.name "あなたの名前"` 等）。未設定だとコミットできない。
- Docker Desktop（または Docker Engine + Compose）がインストール済みであること
- リモートリポジトリ（GitHub 等）が作成済みで、本マシンで `git remote origin` が設定済みであること
- **push 確認**: 本マシンで `git push -u origin main` と `git push -u origin develop` を実行し、GitHub 認証で push できることを確認すること。**パスワード認証は GitHub で廃止されているため、Personal Access Token または SSH が必要**。エラー時は → **devnotes/202503021600-github-auth-sourcetree.md** を参照

## 本マシンで実施する手順

### 1. リポジトリの最新化（既に clone 済みの場合）

```bash
cd "プロジェクトのパス/yakage-italy-veg-site"
git fetch origin
git checkout main
git pull origin main
git checkout develop   # 存在すれば
git pull origin develop
```

### 2. 環境変数ファイルの作成

```bash
cp .env.example .env
# .env を開き、MYSQL_ROOT_PASSWORD, MYSQL_PASSWORD 等を設定（開発用の任意の文字列で可）
```

- 本番と同じパスワードは使わないこと。
- `.env` は Git にコミットしない（.gitignore 済み）。

### 3. Docker コンテナの起動

```bash
docker-compose up -d
```

- 初回はイメージの取得で数分かかることがあります。
- 起動確認: `docker-compose ps` で 3 コンテナ（db, wordpress, phpmyadmin）が `Up` であること。

### 4. WordPress 初期設定

1. ブラウザで **http://localhost:8080** を開く。
2. 表示された言語を選び「続ける」。
3. データベース接続情報は **docker-compose の環境変数で既に渡しているため、画面では「送信」のみでよい**（必要なら以下を確認）。
   - データベース名: `.env` の `MYSQL_DATABASE`（既定: yakage_italy_veg_db）
   - ユーザー名: `MYSQL_USER`（既定: wp_user）
   - パスワード: `MYSQL_PASSWORD`
   - ホスト: `db`
4. 「インストール実行」→ サイト情報（タイトル・管理者ユーザー・パスワード等）を入力して完了。

### 5. 開発用 URL の統一

- **WordPress サイトURL**: http://localhost:8080
- **管理画面**: http://localhost:8080/wp-admin
- **phpMyAdmin**: http://localhost:8888（ログイン: `MYSQL_USER` / `MYSQL_PASSWORD`）

※ 2台目でも同じポート（8080, 8888）を使う前提。別のポートが必要な場合は `docker-compose.yml` の `ports` を変更し、devnotes に「2台目用のポート」として記載すること。

## 別マシンで実施する手順（2台目）

### 1. リポジトリのクローン

```bash
git clone https://github.com/teppeiSQUAT/yakage-italy-veg-site.git
cd yakage-italy-veg-site
```

- 必要に応じて `git checkout develop` で開発ブランチに切り替え。

### 2. 環境変数ファイルの作成

```bash
cp .env.example .env
# .env を編集し、開発用のパスワード等を設定（本マシンと揃えても、別にしても可。本番と同じにはしない）
```

### 3. Docker コンテナの起動

```bash
docker-compose up -d
```

### 4. WordPress について（2台目）

- `wordpress/html` は Git に含めていないため、**2台目ではコンテナ起動時に WordPress が空のディレクトリに展開される**。
- そのまま **http://localhost:8080** で初期設定ウィザードが表示されるので、本マシンと同様に初期設定を行う。
- コンテンツ・DB を本マシンと揃えたい場合は、本マシンでエクスポート（DB は phpMyAdmin や mysqldump、メディアは `wordpress/html/wp-content/uploads` を zip 等）し、2台目でインポートする。

## トラブルシュート

| 現象 | 確認・対処 |
|------|------------|
| ポート 8080 が使えない | `docker-compose.yml` の `wordpress` の `ports` を例: `8081:80` に変更 |
| DB 接続エラー | `.env` の `MYSQL_*` と `docker-compose.yml` の `environment` が一致しているか確認。コンテナを `docker-compose down` 後 `up -d` で再起動 |
| wordpress/html が空のまま | コンテナ内で WordPress が展開される。`docker-compose exec wordpress ls /var/www/html` で確認。必要なら volume を外して一度起動し、中身を確認 |
