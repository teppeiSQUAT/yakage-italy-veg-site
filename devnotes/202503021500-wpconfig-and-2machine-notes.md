# wp-config と 2台運用の注意事項

- **最終更新**: 2025年3月2日

---

## wp-config.php の扱い

- **リポジトリには含めない**（.gitignore で除外済み）。
- 理由: DB 名・パスワード・`WP_HOME` / `WP_SITEURL` 等が環境ごとに異なるため。
- **各マシンで行うこと**:
  1. 初回 `docker-compose up -d` 後、WordPress 初期設定をブラウザで行うと、`wordpress/html/wp-config.php` が自動作成される。
  2. 2台目で clone しただけでは `wordpress/html` は空なので、`docker-compose up -d` でコンテナが起動すると、コンテナ内で WordPress が展開され、初期設定後に同じく `wp-config.php` が生成される。
- 本番用の設定例が必要な場合は、**リポジトリには `wp-config-sample.php` のようなサンプルのみ**を置き、実際の値は記載しないこと。

---

## 2台間でコンフリクトしやすい・注意すべきファイル

| 種別 | ファイル/ディレクトリ | 注意事項 |
|------|------------------------|----------|
| 設定 | `wp-config.php` | リポジトリに含めない。各環境で生成・編集。 |
| 環境変数 | `.env` | リポジトリに含めない。`.env.example` のみ共有。 |
| DB データ | `db/mysql/` | リポジトリに含めない。永続データは各マシンローカル。 |
| WordPress コア | `wordpress/html/` | リポジトリに含めない。コンテナで展開。 |
| テーマ・プラグイン | カスタムテーマ等 | フェーズ5で「リポジトリ内のディレクトリ」として管理する方針。編集は Git で共有。 |

---

## DB・メディアの「主環境」について

- **推奨**: どちらか1台を「主環境」とし、そこでコンテンツ・投稿・メディアを編集。もう1台は主にコード（テーマ・設定ファイル）用。
- 主環境の DB を 2台目に反映したい場合: 主環境で DB エクスポート（phpMyAdmin または `mysqldump`）→ 2台目でインポート。
- メディア（uploads）を揃えたい場合: 主環境の `wordpress/html/wp-content/uploads` を zip 等で共有し、2台目の同階層に展開（または rsync）。※ `wordpress/html` は Git 管理外のため手動または別ツールで共有。

---

## テーマ・プラグインの編集ルール

- **テーマ・プラグインの編集は、リポジトリに含めるファイルのみ行う**（Git で共有）。
- フェーズ5でカスタムテーマを「リポジトリ内のフォルダ」に配置する構成をとる予定。そのフォルダだけ Git で管理し、`docker-compose` の volume で `wordpress/html/wp-content/themes/xxx` にマウントする、などの運用を検討する。
